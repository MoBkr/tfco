## 4. القرارات الهندسية المهمة

### لماذا الصوت لا يمر عبر n8n؟

بعد تجربة حيّة، تبيّن أن جودة Gemini Live Native Audio (الذي يستخدمه ECIT) لا يمكن مطابقتها عبر n8n + browser STT/TTS:
- **Gemini Live**: < 500ms latency، صوت طبيعي سعودي، يفهم السياق والمقاطعة
- **n8n + Web Speech API**: 2-4 ثواني latency، صوت آلي، لا يفهم المقاطعة

القرار: الصوتي يذهب مباشرة إلى Gemini API (نفس طريقة ECIT)، والشات النصي فقط هو الذي يمر عبر n8n workflow.

**اعتبار أمني:** مفتاح Gemini API مُضمَّن في الكود العام على الدومين tfco.sa. يُوصى بتقييده لاحقاً عبر Google Cloud Console بـ HTTP referrer restrictions (`*.tfco.sa/*`).

### لماذا برومبتان منفصلان؟

- **الشات النصي** يحتاج روابط markdown إلزامية لكل منتج (العميل يضغط ليفتح)
- **الشات الصوتي** يحتاج نطق طبيعي للأرقام + منع قراءة URLs صوتياً
- الفصل يمنع التداخل ويسمح بتحسين كل مسار مستقلاً

### الكتالوج — لماذا ملفان؟

- `PRODUCTS.md` ملخص (اسم + سعر + رابط) — كافٍ لـ 90% من الاستفسارات
- `PRODUCTS_DETAILED.md` مفصّل — يُستخدم في RAG vector store للاستفسارات التقنية العميقة (مواصفات دقيقة، أبعاد، طاقة كهربائية...)
